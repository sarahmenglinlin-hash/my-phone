## 1. 产品概述

### 1.1 产品定义
小手机是一款基于Web技术的AI陪伴应用，核心功能是通过实时对话与AI人物进行情感互动。用户可以自定义或导入不同的人物卡，与之进行1对1的陪伴聊天，满足用户的情感陪伴和社交需求。

### 1.2 核心价值
- **Tier 1（核心）**: 人物设定的准确执行，确保AI人物不会OOC（出戏）
- **Tier 2（重要）**: 简洁透明的Apple风格界面
- **Tier 3（便利）**: AI生成人物卡、自动备份等功能（后续版本）

### 1.3 目标用户
- **用户画像**: 18-30岁，有聊天情感需求的男性和女性用户
- **日活预估**: 日活几十人
- **使用场景**: 日常陪伴聊天、情绪倾诉、角色扮演互动

### 1.4 技术方案
- **前端框架**: Vue.js + HTML + CSS + JavaScript
- **数据存储**: IndexedDB (聊天记录、人物卡、世界书) + LocalStorage (配置信息)
- **部署方式**: Web App，支持PWA装到手机桌面
- **后端集成**: 纯本地离线，调用用户自配置的LLM API

  - 保存配置到LocalStorage

#### 2.2.2 【人物捏捏】人物卡创建应用

**进入路径**: 底部导航栏 > 人物捏捏

**页面设计**: 两个入口

##### 入口1: 新建人物
- 用户输入: 人物名字
- 点击"创建"后进入编辑页面
- 跳转到"聊天建模设置页"进行详细配置

##### 入口2: 导入人物
- 支持文件类型: **JSON** 或 **PNG图片** (SillyTavern格式)
- 上传后自动解析，覆盖或重命名处理
- 导入完成后直接加入人物列表

**人物卡数据结构**:
```
{
  id: "unique_id",
  name: "人物名字",           // 必填
  nickname: "备注名",          // 可选，用户自定义，char读不到
  personality: "人物人设",      // 必填
  worldbooks: [              // 关联的世界书
    { id: "wb_1", name: "世界设定" }
  ],
  createdAt: "timestamp",
  updatedAt: "timestamp"
}
```

**数据存储**: IndexedDB

**UI风格**: 卡片式，简洁透明设计

#### 2.2.3 【聊聊】聊天应用（核心功能）

**进入路径**: 底部导航栏 > 聊聊

**页面设计**:
1. **对话列表页**: 显示所有人物卡
   - 人物头像 (可选)
   - 人物名字
   - 最后一条消息预览（可选）
   - 最后更新时间

2. **聊天页**: 1对1对话界面
   - 对话历史显示
   - 底部输入框 + 发送按钮
   - 流式输出显示（逐字显示，若开启）

**核心逻辑 - 人物设定遵守（Tier 1）**:

##### 数据流:
```
用户输入消息 
  ↓
提取关键词 
  ↓
检索世界书（语义近似）→ 获取相关条目
  ↓
构建系统提示词:
  - 人物名字 + 人物设定 (提示词嵌入)
  - 相关的世界书内容 (RAG插入，无上限，根据模型能力)
  - 选定的CoT预设
  - 对话历史 (完整保留)
  ↓
发送到LLM API
  ↓
流式显示回复
  ↓
保存到聊天记录 (IndexedDB)
```

**关键字段说明**:

| 字段 | 作用 | 来源 |
|-----|------|------|
| 人物人设 | 提示词嵌入，确保人物性格一致 | 人物捏捏 App |
| 世界书内容 | RAG检索注入，提供背景知识 | 世界书 App |
| CoT预设 | 提示词模板，指导模型思考链 | 预设 App |
| 对话历史 | 维持对话上下文 | 聊天记录 |

**数据存储**: 
- 聊天记录: IndexedDB
- 临时会话: 运行时内存

**UI风格**: 消息气泡式，简洁透明设计

#### 2.2.4 【世界书】背景故事管理应用

**进入路径**: 底部导航栏 > 世界书

**页面设计**:
1. **世界书列表页**
   - 右上角: "+"按钮（新建或导入）
   - 列表显示已有的世界书
   - 支持删除/编辑

2. **新建/编辑页**
   - 条目名称 (Name)
   - 条目分类 (Category) - 可选，便于组织
   - 条目内容 (Content)
   - 注入位置 (Priority) - 数值越高优先级越高
   
3. **导入页**
   - 支持文件类型: **JSON** 或 **TXT**
   - 批量导入条目

**世界书数据结构**:
```
{
  id: "unique_id",
  name: "世界书名称",
  entries: [
    {
      id: "entry_1",
      title: "条目名称",
      category: "分类",
      content: "条目内容...",
      priority: 10,           // 注入优先级（越高越优先）
      keywords: ["关键词1", "关键词2"]  // 自动或手动提取
    }
  ],
  createdAt: "timestamp",
  updatedAt: "timestamp"
}
```

**关键机制 - RAG语义检索**:
- 用户发送消息时，系统自动提取关键词
- 对世界书所有条目进行**语义相似度匹配**（不是精准词匹配）
- 根据优先级排序，将相关条目注入到LLM提示词中
- **无上下文大小限制**：根据模型自身能力自动调整注入量

**数据存储**: IndexedDB

**UI风格**: 条目卡片式，简洁透明设计

#### 2.2.5 【预设】CoT提示词预设应用

**进入路径**: 底部导航栏 > 预设

**页面设计**:
1. **预设列表页**
   - 右上角: "+"按钮（新建或导入）
   - 当前使用的预设高亮显示
   - 全局只能使用**一个**预设（Agent跟随CoT走）
   - 支持删除/编辑/切换

2. **新建/编辑页**
   - 预设名称
   - 预设内容 (CoT提示词模板)
   
3. **导入页**
   - 支持文件类型: **JSON**
   - 导入自SillyTavern或其他社区预设

**预设数据结构**:
```
{
  id: "unique_id",
  name: "预设名称",
  content: "{{char}} 思考链条：\n1. 理解用户意图\n2. 检查人物设定\n3. 生成回复...",
  isActive: true,           // 是否为当前使用的预设
  createdAt: "timestamp",
  updatedAt: "timestamp"
}
```

**使用逻辑**:
- 用户选择一个预设后，所有聊天调用都会使用该CoT
- 切换预设：点击"使用此预设"即可切换
- Agent会严格遵循当前CoT的思考链条进行回复

**数据存储**: IndexedDB

**UI风格**: 卡片式，简洁透明设计

### 2.4 导出/导入功能

**进入路径**: API 设置页的"备份与恢复"模块

**功能**:

#### 导出功能
- [ ] 导出全部数据 (ZIP文件)
  - 所有人物卡
  - 所有世界书
  - 所有预设
  - 全部聊天记录
- [ ] 按需导出 (选择导出的数据类型)

#### 导入功能
- [ ] 选择导入文件（JSON or ZIP）
- [ ] 冲突处理: 
  - 覆盖现有数据
  - 合并 (保留现有数据，新增导入数据)
  - 取消

**工作流**:
```
首次使用:
  App启动 → 提示导入 (可选) → 开始使用

日常使用:
  本地存储自动保存 (IndexedDB)

定期备份:
  API页 → 备份与恢复 → 导出全部 → 保存到设备

数据恢复:
  APP启动 → 手动导入 → 选择文件 → 恢复成功
```

## 3. 非功能需求

### 3.1 UI/UX设计方向
- **风格**: Apple iOS简洁透明风格
- **色调**: 浅色为主，支持深色模式 (后续)
- **响应式**: 适配手机、平板、桌面宽度
- **字体**: 系统字体 (San Francisco, -apple-system等)
- **间距**: 充足的留白，简洁不拥挤

### 3.2 性能要求
- 首屏加载时间 < 2s
- 消息响应延迟 < 100ms (UI响应)
- IndexedDB查询 < 50ms
- 支持离线访问 (通过PWA)

### 3.3 数据安全
- 所有数据本地存储，不上传服务器
- 用户API Key仅存LocalStorage，不传输
- 聊天记录仅存本地IndexedDB
- 支持用户手动导出备份

### 3.4 浏览器兼容性
- Chrome / Edge (推荐)
- Safari (iOS 15+)
- Firefox (最新版)
- PWA支持 (可装到手机桌面)

```
  ↓
显示响应 ✓
```

### 7.2 术语表

| 术语 | 定义 |
|-----|------|
| OOC | Out Of Character，出戏，人物行为不符合设定 |
| RAG | Retrieval Augmented Generation，检索增强生成 |
| CoT | Chain of Thought，思维链提示 |
| IndexedDB | 浏览器本地数据库 |
| LocalStorage | 浏览器键值对存储 |
| PWA | Progressive Web App，渐进式网页应用 |
| Token | LLM的最小输入单位 |
